Estrategias de Aprendizado para Descoberta Coletiva

Bandit

Multi-Armed Bandit por Posicao

Reforca cada token usado proporcionalmente ao reward recebido. Simples e rapido, mas trata todas as posicoes igualmente.

Probabilidades por Posicao
Vantagens
  • Simples de implementar
  • Convergencia rapida inicial
  • Baixo custo computacional
Desvantagens
  • Fica preso em maximos locais
  • Nao diferencia posicoes
  • Reforca erros junto com acertos

Single Var

Feedback Comparativo

Compara tentativas consecutivas para inferir quais posicoes estao corretas. Se mudar uma posicao melhora, a anterior estava errada.

Vantagens
  • Identifica posicoes corretas
  • Aprende com erros
  • Mais eficiente que bandit
Desvantagens
  • Precisa de historico
  • Pode confundir com ruido
  • Mais complexo

Explorer

Epsilon-Greedy com Decay

Alta exploracao no inicio, foca depois. Adiciona ruido que diminui com o tempo, escapando de maximos locais.

Exploracao ao Longo do Tempo
ε → 0
Vantagens
  • Escapa de maximos locais
  • Balance exploracao/explotacao
  • Robusto
Desvantagens
  • Mais lento no inicio
  • Parametros a ajustar
  • Pode explorar demais

📐 Technical Details

Bandit Strategy

Update Rule:

P_new[pos, token] += α * reward
P_new = normalize(P_new)  # Each row sums to 1

Complexity: O(L) per update, where L = sequence length

Parameters:

  • α (learning_rate): 0.05-0.15 recommended. Higher = faster but less stable

When to use: Quick testing, small search spaces, when speed > quality

Single Var Strategy

Update Rule:

# Analyze history to compute confidence[pos]
for pos in changed_positions:
    if reward improved: confidence[pos] += 0.1
    if reward worsened: confidence[pos] -= 0.1

P_new[pos, token] += α * reward * confidence[pos]
P_new = normalize(P_new)

Complexity: O(L + H) per update, where H = history size (typically 10)

Parameters:

  • α (learning_rate): 0.10-0.20 recommended
  • max_history: 5-15 attempts (default: 10)

When to use: Medium/large search spaces, when you need efficiency, HIVE mode

Explorer Strategy

Update Rule:

P_new[pos, token] += α * reward
noise = uniform(0, ε, shape=P.shape)
P_new += noise

ε *= decay  # Typically 0.995
P_new = normalize(P_new)

Complexity: O(L * M) per update, where M = alphabet size

Parameters:

  • α (learning_rate): 0.05-0.12 recommended (lower than bandit due to noise)
  • initial_epsilon: 0.2-0.4 (higher = more exploration)
  • decay: 0.990-0.999 (slower decay = longer exploration)

When to use: Complex landscapes, LOCAL mode, when agents get stuck, long runs

💡 Practical Guidelines

  • Start with Bandit for prototyping and quick tests
  • Use Single Var for production HIVE mode (best efficiency/quality tradeoff)
  • Use Explorer for LOCAL mode or when diversity is critical
  • Learning Rate Sweet Spot: 10-20% (0.10-0.20 in code)
  • Too high α: Oscillation, instability
  • Too low α: Slow convergence, wasted energy
